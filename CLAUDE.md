```markdown
# Обзор проекта Online Store

## Назначение проекта
Учебный проект интернет-магазина (Exam Project 2025). Проект реализует систему с разделением прав доступа (Admin, Manager, Customer), логикой управления заказами, выбором способа оплаты (через паттерн Strategy: карта, кошелек, СБП) и автоматической выгрузкой аудиторских отчетов в формате CSV. Основная часть бизнес-логики (например, пересчет сумм заказа и логирование статусов) реализована на уровне базы данных.

## Стек
- **Язык**: C++17
- **База данных**: PostgreSQL
- **Зависимости**: libpqxx (C++ клиент для работы с PostgreSQL)
- **Система сборки**: CMake (минимум 3.16)

## Структура каталогов
- `include/` — заголовочные файлы (`.h`).
- `src/` — файлы исходного кода (`.cpp`).
- `sql/` — скрипты инициализации и логики базы данных (таблицы, функции, процедуры, триггеры, начальные данные).
- `reports/` — директория для сохранения генерируемых отчетов (создается автоматически приложением).
- `REVIEW.md` — документ с результатами ревью проекта (замечания по архитектуре, БД, безопасности и рекомендации по рефакторингу).

## Команды сборки/тестов/линтера
- **Сборка**: Используется стандартный процесс CMake. Конфигурация включает специфичные пути для поиска `libpqxx` через Homebrew на macOS. Имя собираемого бинарника — `online_store`.
  - Примеры стандартных команд: `cmake -B build` и `cmake --build build`.
- **Тесты**: В репозитории не настроены фреймворки для тестирования (GTest, Catch2 и интеграция с CTest отсутствуют).
- **Линтеры**: Конфигурационные файлы линтеров или форматтеров (такие как `.clang-format` или `.clang-tidy`) отсутствуют.

## Конвенции кода
- **ООП и Паттерны**: Активное использование наследования, полиморфизма (интерфейсы `User`, `PaymentStrategy`), а также композиции и агрегации (сохранение заказов в пользователе).
- **Управление памятью**: Исключительное использование умных указателей (`std::unique_ptr`, `std::shared_ptr`) вместо сырых.
- **Алгоритмы и STL**: Применение алгоритмов стандартной библиотеки (`std::copy_if`, `std::find_if`, `std::accumulate`) в связке с лямбда-выражениями.
- **Шаблоны**: Использование шаблонов (например, `DatabaseConnection<T>`) для универсализации доступа к данным.
```
